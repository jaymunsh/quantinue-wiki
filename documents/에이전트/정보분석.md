# 📰 정보분석 에이전트 (⑤⑥ 공시·뉴스)

!!! note "⚪ 설계 예정 · 담당 정창욱"
    한 에이전트가 공시·뉴스를 함께 분석(속도차·상호보완). SQLite 기반 코드가 일부 있으나 위키 페이지는 계약 기준. 코드 정리되면 채운다.

## 역할

오늘의 50 종목의 **공시(8-K·10-Q)와 뉴스**를 LLM으로 분석해 호재/악재·중요도·신뢰도를 산출. 찌라시·루머를 거르는 **신뢰도 필터**가 핵심.

- **입력:** `tb_daily_pick` 50 티커 + SEC 공시·실시간 뉴스
- **출력:** `tb_disclosure`(공시) · `tb_news`(뉴스) — event_type 11종 공유
- **원칙:** 코드=사실 / LLM=해석 · hard_block(파산·상폐 등) 코드 강제 · 시계열 누적(새 건마다 append)

## 계약·결정

- 스키마: [데이터 계약](../facts/데이터계약.md) `tb_disclosure`·`tb_news`·event_type 온톨로지
- 관련: [회의 안건](../질문.md) B2(category JOIN)·B3(cross_source_confirmed)·Bundle 감축(24→18/31→28)
- 회의: [1차](../회의록/2026-06-30.md)(신뢰도 필터·소셜감성 2차)·[3차](../회의록/2026-07-03.md)(확정도 1.0·confidence)

> 코드가 정리되면: 수집·분석 파이프라인·신뢰도 필터·집계 공식으로 확장.

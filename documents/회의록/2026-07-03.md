# 🗣️ 회의록 2026-07-03 (3차 · 데이터·인터페이스)

!!! abstract "3줄 요약"
    공시·뉴스의 필드(확정도·confidence·문턱)와 DB 테이블 구조를 화면공유하며 맞췄다. 유니버스↔데일리픽 순서, 안정/공격 문턱 기준을 조율. 대부분 데이터 계약·tb_ 접두사 결정으로 수렴됨.

## ✅ 결정된 것 (대부분 이미 반영됨)

| 결정 | 반영 |
|---|---|
| 공시 **확정도(confirmed_score)=1.0 고정**(공식문서라 항상 확정) · confidence(LLM 자기확신) 분리 | 데이터 계약 tb_disclosure |
| 문턱값이 안정형/공격형 별로 다름 — "모두 만족해야 통과, 아니면 0" | 데이터 계약 §2 (이후 공격형 단일로 단순화) |
| DB 테이블 전체 관계도(ERD) 정리 · 유니버스는 데일리픽 **앞**에 위치 | 데이터 계약 ERD |
| 회고 기록 = 판단 근거·요약을 자세히 남김 | tb_review lesson |

## 1. 한 사이클 — 10단계

매일 08:30 AM ET에 한 사이클이 돈다. 한 사이클 = 1개 `cycle_id`로 묶임.

```
① 스크리너      시총 top2000 → 오늘의 후보 N개                [지현·규칙]
② 기술 분석     후보별 지표(trend·RSI·MACD·ATR)              [지현·규칙]
③ 매크로        시장 국면(regime·risk_score) 1행             [지현·데이터]
④ 공시 분석     후보만 SEC 공시 해석 → Bundle                [창욱·LLM]
⑤ 뉴스 분석     후보만 뉴스 감성·신뢰도 → Bundle             [창욱·LLM]
        │  (① ~ ⑤ 신호를 각자 DB에 저장)
        ▼
⑥ Strategist    신호 종합 → 매수 / 보류(NO_TRADE) + 근거      [은미·LLM]
⑦ Risk Critic   적대적 반박 → 통과 / 기각                     [미연·LLM]
⑧ Portfolio Mgr 수량·비중·EV 계산                            [지현·코드]
⑧' 리스크 게이트 손절·한도·하드룰 → 허용/차단/축소            [지현·규칙]
⑨ 실행          PaperBroker 가상 체결 + 근거 기록             [지현·코드]
⑩ Reviewer      결과 회고 → tb_memory_entries (observe-only)  [성혁·LLM]
        │
        └─(회고 메모리가 다음 사이클 ⑥ Strategist로 피드백)
```

## 2. 확정된 핵심 원칙

| 원칙 | 내용 |
|---|---|
| **Macro-first / 후보 압축** | 시총 top2000 → 스크리너로 후보만 추림 → **공시·뉴스는 후보에만** LLM 분석 (비용 방어) |
| **Pull 방식** | 각 팀이 신호를 DB에 저장(INSERT) → Strategist가 필요한 필드만 읽음(SELECT). 택배(push) 아님 |
| **코드 게이트 샌드위치** | LLM은 "제안·해석"만. 앞문(hard_block·macro_veto)·뒷문(min_conviction·리스크게이트)은 코드가 강제 → LLM 폭주 차단 |
| **NO_TRADE는 정상 결과** | "안 사는 판단"도 근거와 함께 저장 |
| **전부 가상** | PaperBroker 가상 체결. 실계좌·실주문·증권사 API 없음 |
| **cycle_id로 재현** | 한 사이클의 모든 산출물이 같은 cycle_id → 나중에 "왜 이렇게 판단했나" 완전 재현 |

## 3. 단계별 소유자 (누가 만드나)

| 단계 | 담당 | 타입 | 출력 테이블 |
|---|---|---|---|
| ① 스크리너 · ② 기술 · ③ 매크로 | 김지현 | 규칙/데이터 | tb_daily_pick · tb_technical_signals · tb_macro_signals |
| ④ 공시 · ⑤ 뉴스 | 정창욱 | LLM | tb_disclosure_signals · tb_news_signals |
| ⑥ Strategist | 이은미 | LLM | tb_strategist_signals |
| ⑦ Risk Critic | 김미연 | LLM | tb_critic_verdict |
| ⑧ PM·게이트 · ⑨ 실행 | 김지현 | 코드/규칙 | tb_portfolio_decision · tb_order · tb_fill |
| ⑩ Reviewer | 문성혁 | LLM | tb_memory_entries |

> 각 테이블의 필드·타입은 **`데이터 계약`** 페이지 참조.

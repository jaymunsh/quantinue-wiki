★ Scoring

0~1 지표 산출 근거 (핵심)

모든 점수는 0~1 소수 2자리. 아래가 "이 숫자가 어떻게/왜 나오는가"의 단일 진실원천. 코드: app/common/analysis_bundle.py.

6-1. 정규화 · 상수

항목규칙값
_n10(v)LLM 0~10 척도 → 0~1clamp(v,0,10)/10, 2자리
certainty → confidence모호성 수치화High 0.9 · Medium 0.6 · Low 0.3 (없으면 0.5)
grade_score 매핑출처 정책등급→점수ALLOW 1.0 · GRAY 0.6 · BLOCK 0.0

6-2. sentiment_score — 방향(0=강악재·0.5=중립·1=강호재)

비대칭 신뢰 감쇠: 호재(+)만 confidence로 중립쪽 감쇠, 악재(−)는 유지(경고를 흐리지 않는 보수 원칙).

# signed = 방향부호 × 강도(-1~+1)
if signed > 0:                 # 호재만 감쇠
    signed = signed × confidence
sentiment_score = 0.5 + 0.5 × signed

_senti_score(+1.0, conf=1.0) = 1.00   # 확신 호재
_senti_score(+1.0, conf=0.2) = 0.60   # 저신뢰 호재 → 중립쪽으로 눌림
_senti_score(-1.0, conf=0.2) = 0.00   # 저신뢰 악재 → 감쇠 없음

6-3. 뉴스 신뢰가중 집계 (여러 건 → 대표/점수)

기사별 신뢰도(가중치)로 가중 평균한다. "루머 10건 < 확정 1건".

# 기사별 가중치 (_news_conf)
weight = certainty(.9/.6/.3) × (1.0 if is_confirmed else 0.5) × source_trust(0~1)
# ★ source_trust=0.0(무신뢰)을 1.0으로 오인하지 않음 (None만 1.0 폴백)
결과 필드산출식 · 근거
importance부호 없는 강도의 신뢰가중 평균 — 방향과 독립. 호·악재 섞여도 상쇄되지 않아 같은 강도면 항상 같은 값. (전부 저신뢰=가중치 0이면 등가중 폴백으로 붕괴 방지)
peak_importance집계 전 최고 강도 max(_n10(importance))
agg_signed(방향)기사별 강도×부호의 신뢰가중 평균. 단 강한 악재(min ≤ −0.7) 1건이라도 있으면 그 값 채택(보수)
sentiment_score_senti_score(agg_signed, confidence)
sentiment(라벨)sentiment_score에서 파생(라벨↔점수 항상 일치): 강한악재→negative · 호·악재 각 신뢰가중 ≥30%→mixed · 그외 >.5=positive/<.5=negative/=.5=neutral
risk_score·source_trust·grade_score기사별 값의 신뢰가중 평균
confidence기사별 가중치의 단순 평균(루머 많으면 급락)
confirmed_score확정 비율 = news_confirmed / news_count
대표 기사importance × weight(신뢰가중) 최대 → 시끄러운 루머가 헤드라인 안 되게
공시 점수 — 단일 신호라 집계 없음. importance=_n10(LLM) · confidence=certainty매핑 · confirmed_score=1.0(법적 제출=사실) · sentiment_score=_senti_score(부호×importance, confidence).

6-4. fact_check — 팩트체크 판정(코드, LLM 무관)

판정조건(경계)
공시 Verified항상 — "Verified — official SEC filing (legally binding fact)."
Verified뉴스: ratio ≥ 0.5 그리고 source_trust ≥ 0.6 (ratio=확정/전체)
Unverified (rumor)confirmed == 0 또는 source_trust < 0.4
Partially verified그 외

Boundary

코드 vs LLM — 무엇을 누가 결정하나

핵심 원칙: 코드 = 사실, LLM = 해석만. 텍스트에서 숫자 추출은 환각 위험이라 금지. 안전장치는 코드가, 뉘앙스는 LLM이.

코드 가 확정

  • 수치(XBRL 매출·EPS·YoY) · Form4 스코어링 · Item 기본값
  • 하드리스크 키워드 → 권한 강제
  • 출처 등급 · 키워드 사전필터 · 최종권한 종합
  • 모든 0~1 지표 계산 · fact_check · 신뢰가중 집계 · 대표 선정
  • 증분 판정 · 저장 · 전방수익률

LLM 은 해석만

  • 8-K 본문/뉴스 헤드라인의 event_type·sentiment·importance(0~10)·risk_score
  • is_confirmed(확정/추측) · source_trust(0~1)
  • reason · verdict · summary · keywords (영어)
  • 추론강제: reasoning(CoT) → certainty_level 순서로 사고

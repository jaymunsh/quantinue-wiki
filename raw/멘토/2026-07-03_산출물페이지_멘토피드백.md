## 📌 제출 규칙 (공통)

1. **단계별로 제출한다.** 각 단계의 `필요 산출물`을 모두 채우고 `완료 기준`을 스스로 점검한 뒤 멘토 확인을 받는다.
2. **산출물 올리는 법** — 각 단계 `📎 제출물` 아래에 파일을 직접 드래그하거나, GitHub·문서·Figma 링크를 붙여넣는다.
3. **멘토 피드백은 각 단계 안에서 받는다.** 단계마다 `🗒️ 멘토 피드백` 영역에 코멘트가 달린다.

## 0️⃣ 일정 계획 (간트차트)

**필요 산출물**

- [ ]  전체 일정 간트차트 — 기획→설계→디자인→개발→테스트 배치 (그리는 도구는 자유: 타임라인·Mermaid·이미지 등)
- [ ]  각 단계를 세부 태스크로 분해 + 태스크별 소요 기간(일수) 추정
- [ ]  태스크 간 의존성(선행 태스크)과 테스트·버퍼 기간 반영

**완료 기준** — 전체 합산이 마감 안에 들어오고, 각 태스크의 기간·의존성·버퍼가 한눈에 드러나는가

> 📎
> 

**🗒️ 멘토 피드백** *(멘토 작성)*

> 
> 
> 
> 현재 안내드린 일정보다 앞당겨 진행되고 있는 점은 좋습니다. 다만 구현 단계도 조금 더 세분화하여 작성하면 일정을 파악하기 쉬울 것 같습니다. (예: A 에이전트 개발, B 에이전트 개발, 화면 구현 등)
> 
> 또한 **'결정저널 프론트'**가 어떤 작업을 의미하는지 다소 모호한데, 구체적으로 표현해 주시면 좋겠습니다.
> 
> 마지막으로 발표 PPT는 가능하면 발표 당일보다는 **전날까지 완성**하는 것을 목표로 일정을 조정해 보시면 좋을 것 같습니다.
>

## 1️⃣ 기획

**필요 산출물**

- [ ]  벤치마킹 분석서 또는 기획서
- [ ]  요구사항 정의서

**완료 기준** — 벤치마킹을 바탕으로 사용자 문제를 정의하고, 요구사항이 우선순위에 따라 정리되었는가

> 📎
> 

**🗒️ 멘토 피드백** *(멘토 작성)*

> 
> 
> 
> 전반적으로 서비스 방향과 구현 범위가 명확하게 정리되어 있습니다. 특히 PaperBroker 기반의 가상 투자 환경을 중심으로 시스템 파이프라인과 요구사항을 일관성 있게 구성한 점이 좋았습니다.
> 
> 다만 현재는 **MVP 성공 기준**이 정의되어 있지 않아 어느 수준까지 구현되면 프로젝트 목표를 달성했다고 볼 것인지 판단하기 어렵습니다.
> 
> 후보 선정 → 분석 → 의사결정 → 가상 체결 → 결정 저널 기록 등 핵심 기능을 기준으로 **MVP 완료 조건을 구체적으로 정의**해 주시면, 이후 설계와 테스트 기준도 함께 명확해질 것 같습니다.

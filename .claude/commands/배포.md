description: 검증 후 배포 — main push 시 GitHub Actions가 자동 배포 (jaymunsh.github.io/quantinue-wiki)
---

위키를 배포하라. **배포 = main에 push** (`.github/workflows/deploy.yml`이 lint → strict build → gh-deploy 자동 수행).

1. `python3 tools/lint.py` — ERROR 0 확인. ERROR 있으면 **중단**하고 보고.
2. `python3 -m mkdocs build --strict` — 로컬 선검증 (CI와 같은 조건).
3. 미커밋 변경이 있으면 보고하고 커밋 여부를 확인받는다 (논리 단위별 분리, 한글 메시지).
4. `git push origin main` → Actions가 배포. `gh run watch` 또는
   `gh run list --workflow=deploy-wiki --limit 1` 로 성공 확인.
5. 라이브 확인: `curl -s -o /dev/null -w "%{http_code}" https://jaymunsh.github.io/quantinue-wiki/` 가 200 (반영 1~2분).
6. 배포된 커밋 해시와 함께 결과 보고.

**문제 시:**
- Actions 실패 → `gh run view --log-failed` 로 원인 확인 (lint/strict가 CI에서만 실패하면 로컬 환경 차이).
- Pages가 Queued에 멈춤 → Actions 탭에서 `deploy-wiki` 수동 재실행(workflow_dispatch), 또는 gh-pages에 빈 커밋 push.
- CI 불능(긴급) → 수동 폴백: `python3 -m mkdocs gh-deploy --force` (운영자 확인 후).

    🗣️ 회의록 2026-07-06 (4차)¶

    3줄 요약

    통합 데이터 스키마를 화면공유하며 실행 정책을 확정 — 1차엔 매도 없이 매수/보류만, 보류도 전량 회고 기록. 어닝·FOMC 가드레일(코드 하드 차단)과 실시간 시세(API 배치) 방식을 합의. 스케줄 주기·손절 수치는 추가 논의.

    ✅ 결정된 것 (병입 대기)¶

    결정 관련 근거 발언 요지
    1차 매도 미사용 — Strategist는 매수/보류만 Critic에 넘기고, 매도는 포트폴리오의 자동 손절 규칙으로만 파이프라인 ⑦⑨ "일단 매도를 쓰지 마세요… 매도는 포트폴리오에 얼마에 팔지 적어놓고 그 기준으로만"
    보류(NO_TRADE)도 50종목 전량 회고 기록 B11-④ · tb_review "보류도 넣어야 돼. 실시간 시세로 회고 쓰거든요"
    어닝·FOMC 가드레일 = 코드 하드 차단 — 예정된 실적발표·FOMC 임박 종목은 무조건 매수 차단(발표 시 10~20% 급변 위험). 리스크게이트 위 이중 안전망 파이프라인 ⑨ (신규) "날짜가 정해져 있어 에이전트 아닌 코드로. 어닝 직전이면 무조건 차단"
    실시간 시세 = API 배치 조회(비저장) — 50종목을 판단 시점에 배치 호출(현재가·52주 최고/최저). DB 저장 안 함(넣는 순간 과거값). Critic도 이 시세 참조 tb_strategist_signals 조율① "시세는 넣을 때마다 과거값… 스케줄러가 확인할 때마다 배치 50개 호출"
    페르소나 1차 미적용 (2차) tb_strategist_signals "페르소나 일단 안 넣기로"

    ⏸️ 보류·계속 논의¶

    안건 경과·이유
    Strategist 스케줄 주기 1차엔 4시간 정도로 여유롭게 제안 → 확정 아님. "10분은 과하고 4시간 정도, 나오는 거 보고 조정"
    손절 라인 수치 발언에 "-5% 이하 매도"[?]가 나왔으나 공격형 정책(−15%)과 상이 → 확인 필요
    개인별 시드 차이 반영 시드머니에 따라 종목이 달라질지(소수점 매매라 비율 동일 가능) → "해봐야 안다"

    📋 액션 아이템¶

    담당 할 일 기한
    지현 실시간 시세 조회 가이드라인 은미·미연에 전달 · 스케줄 주기 확정 미정
    은미 tb_strategist_signals에 실시간 시세 필드 반영 · side에서 매도 제외 미정
    성혁 회고에 보류 포함 반영 (완료 — tb_review) —

    💬 논의 맥락¶

    • 왜 매도를 뺐나: 매도 자율 허용은 리스크가 크고, 손절은 포트폴리오가 매수가 대비 규칙으로 자동 처리 → Strategist는 매수 제안만.
    • 왜 보류도 기록: "안 샀는데 올랐다/떨어졌다"가 회고에 남아야 보수적 판단의 질을 평가 가능.
    • 통합 스키마(창욱·은미 등 병합, 예시 데이터 포함)를 화면공유하며 진행.

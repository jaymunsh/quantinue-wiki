    🗣️ 회의록 2026-07-09 (오전 · ERD 정합 1부)¶

    3줄 요약

    창욱의 공시·뉴스 테이블을 컬럼 단위로 함께 훑는 정합 회의. 공시 쪽은 요약본·근거·PK 등 방향이 잡혔으나, 뉴스 "대표기사 선정 기준"에서 막혀 뉴스 전체가 미결로 넘어감(오후 2부·다음 회의로). STT 잡음 큼 — 아래는 확정 아닌 논의 경과.

    전사 품질 낮음

    자동 음성인식(STT) 원본이라 오인식 많음. 표현이 불명확한 곳은 [?]. 확정으로 읽지 말 것 — 결정은 병입 대기.

    ✅ 결정된 것 (병입 대기 → 확정되면 결정 로그 # 부여)¶

    결정 관련 안건 근거 발언 요지
    공시 summary = 원문이 아니라 LLM 요약본 B2(공시필드) "원문 100페이지도 있어 다 못 가져옴 → 핵심 요약으로"
    event_type(사건 11종)에 판단 근거(reason) 신설 B2 "importance를 왜 그렇게 줬는지 근거가 있어야 — 지금은 sentiment 근거만"
    저장 시각 = 한국 시간(KST) 기준 통일 [?] 신규 "수집은 미국 시각이라도 저장·조회는 한국시간이 보기 편함"(시세는 미국 유지 여지)
    센티멘트 = 텍스트 라벨 빼고 sentiment_score 하나로 통합 (0=악재·0.5=중립·1=호재, 소수점 3자리로 강도) [?] B3(뉴스필드) "라벨과 스코어 역할이 겹침 → score 하나로 호·중·악 + 강도"
    테이블마다 고유 PK 필수(분리형 복합키) B9·스키마 "각 테이블에 본인만의 고유 ID가 있어야 — 지금 PK가 엉망"

    ⏸️ 보류·계속 논의¶

    안건 경과·이유
    🔴 뉴스 대표기사 선정 기준 창욱(실내가중치 = 문장 단정성·사실성·출처신뢰) ↔ 은미가 준 파일 방식 불일치. "대표"의 정의 자체가 애매 → 시키는 사람이 이해 못 하면 에이전트도 못 함. 이게 안 잡히면 하류(07~08) 전체가 흔들림. 다음 회의까지 창욱이 재정리
    뉴스 event_type(사건종류) 중복 뉴스 event_type ↔ top_evidence(대표근거)가 겹치는지 · 공시와 매칭될 때만 채우고 아니면 null? — 미결
    뉴스 원본 저장 테이블 신설 5분마다 전 종목 뉴스를 다 담는 raw 테이블 + 50 데일리픽 매칭·집계 테이블 2단이 필요하다는 지적. 창욱은 집계 1행만 설계 → 원본 테이블 논의
    공시 PK = 접수번호(filing_no)? 접수번호에 영문 혼입 가능성 → 창욱 확인 후 결정

    📋 액션 아이템¶

    담당 할 일 기한
    창욱 공시·뉴스 파일 2개로 분리, 각 상단에 ERD + 컬럼을 순서대로 설명, event_type 11종 반영, importance reason 신설, PK 정리, HTML로 제공 다음 회의 전
    창욱 대표기사 선정 기준을 데이터 예시와 함께 "이렇게 모아 이 점수로 이게 대표" 식으로 재설명 다음 회의

    💬 논의 맥락¶

    • 반복된 호소: "컬럼이 흔들리면 모두가 흔들린다" — 뒷단(은미·미연)이 이 컬럼으로 로직을 짜야 하므로, 스키마가 안 잡히면 에이전트 구현이 계속 뒤집힘.
    • 클로드가 자동 생성한 컬럼이 많아 "진짜 필요한 것"이 흐려짐 → 창욱이 "내가 필요한 것"부터 정의하고 줄이자는 방향.
